# PRD: Персональный сайт-канвас (портфолио, блог, резюме)

Дата: 2026-07-08
Статус: на утверждении
Референсы: bilal.world (механика канваса), панель-инспектор дизайн-инструмента (стиль), мультфильмы 1930-х rubber-hose (декор)

## 1. Обзор

Персональный сайт Виталия в виде бесконечного канваса, по которому расставлены фреймы-страницы (Обо мне, Проекты, Блог, Резюме). Посетитель панорамирует и зумирует канвас как в Figma, перелетает к разделам через плавающую нижнюю навигацию, ориентируется по мини-карте. Посты блога импортируются из Telegram-канала автоматически.

Цели:

- Запоминающаяся подача: сайт-канвас вместо обычного лендинга.
- Ноль ручной рутины для блога: опубликовал в Telegram — появилось на сайте.
- Простое сопровождение: статика, контент в markdown, без БД и серверов.

## 2. Разделы (фреймы) первой версии

| Фрейм | Содержимое |
|---|---|
| Интро / Обо мне | Приветствие, кто я, чем занимаюсь, контакты, соцссылки |
| Проекты | Карточки проектов из markdown: описание, скриншоты, ссылки, теги |
| Блог | Список постов Telegram-канала; чтение — в боковой панели |
| Резюме | Опыт, навыки, образование + кнопка «Скачать PDF» |

Каждый фрейм имеет фиксированный размер (свой у каждого), задаваемый в конфиге раскладки. Внутри фрейма — обычная HTML-страница: текст, изображения, видео, скрипты, встраиваемые виджеты. Контент фрейма при переполнении скроллится внутри фрейма.

## 3. Канвас-движок

Механика:

- **Панорамирование**: зажатый пробел + драг ЛКМ, либо драг ПКМ (контекстное меню браузера подавляется), либо жест тачпада. Лёгкая инерция после отпускания.
- **Зум**: колесо мыши / щипок тачпада, к точке под курсором. Диапазон 25–150%. Индикатор текущего масштаба в нижней навигации, клик по нему — сброс к 100%.
- **Перетаскивание фреймов**: посетитель может таскать фреймы за любую область, не занятую интерактивным контентом (приоритет — заголовок/рамка). Ресайз фреймов запрещён. Изменения позиции живут до перезагрузки страницы и нигде не сохраняются.
- **Стартовая позиция**: вьюпорт центрируется на фрейме «Интро» при заходе без якоря в URL.
- Реализация: один контейнер с `transform: translate(...) scale(...)`, фреймы позиционируются абсолютно по координатам из `layout.json`.

Производительность:

- Фреймы вне вьюпорта (с запасом в один экран) не рендерят содержимое — виртуализация.
- Тяжёлое содержимое (видео, t.me-виджеты) монтируется лениво, при попадании фрейма во вьюпорт.
- Цель: 60 fps панорамирования на среднем ноутбуке, Lighthouse Performance ≥ 90 для мобильной ленты.

## 4. Навигация

Плавающая панель по центру нижней кромки экрана:

- Пункты: Обо мне, Проекты, Блог, Резюме. Активный подсвечен зелёным.
- Клик — плавный перелёт (анимация pan+zoom, ~600 мс, easing с лёгким overshoot) к фрейму, масштаб подбирается так, чтобы фрейм поместился во вьюпорт с полями.
- Активный пункт определяется автоматически: фрейм, занимающий наибольшую долю вьюпорта.
- Справа за разделителем: индикатор зума и тумблер темы.

## 5. Мини-карта

- Положение: правый нижний угол.
- Появляется при любом движении канваса (пан/зум/перелёт), плавно исчезает через ~1.5 с покоя. При наведении курсора — не исчезает.
- Показывает: уменьшенные прямоугольники всех фреймов (активный — зелёный), рамку текущего вьюпорта.
- Клик/драг по мини-карте — мгновенное перемещение вьюпорта в указанную точку.

## 6. Блог и Telegram

Отображение:

- Фрейм «Блог» — список постов из `content/blog.json`: заголовок, дата, теги.
- Клик по посту — справа выезжает панель (~480 px, поверх канваса, закрытие по крестику/Esc/клику мимо) с официальным embed-виджетом Telegram этого поста (`telegram-widget.js`, `data-telegram-post`, тёмная тема виджета в тёмной теме сайта).
- Прямая ссылка на пост: `#/blog/<id>` открывает сайт с уже открытой панелью.

Автоимпорт (GitHub Action по расписанию, раз в час):

1. Скрипт скачивает публичную веб-версию канала `t.me/s/<канал>`.
2. Парсит новые посты: id, первая строка текста как заголовок, дата.
3. Дописывает новые записи в `content/blog.json`; если изменений нет — завершается без коммита.
4. При изменениях коммитит и пушит — деплой пересобирает сайт.

Правила слияния: записи сверяются по id поста; ручные правки полей (заголовок, теги, `hidden: true`) не затираются; `hidden: true` скрывает пост из списка. Канал должен быть публичным.

Риск: `t.me/s/` — неофициальный интерфейс; при изменении разметки парсер перестаёт находить новые посты (сайт продолжает работать со старым списком), Action сигналит об ошибке — чинится правкой парсера.

## 7. Контент-модель

```
content/
  frames/
    about.md        # текст фрейма «Обо мне» (фронтматтер: title, subtitle)
    resume.md       # резюме
  projects/
    <slug>.md       # по файлу на проект (фронтматтер: title, tags, links, cover)
  blog.json         # посты: [{ id, url, title, date, tags?, hidden? }]
  layout.json       # раскладка: { frameId: { x, y, w, h } }
site.json           # имя, контакты, соцссылки, SEO-мета
public/             # изображения, видео, cv.pdf, ассеты курсора
```

- Markdown конвертируется в HTML при сборке.
- Добавить проект = добавить `.md`-файл и запушить.
- PDF резюме кладётся вручную в `public/cv.pdf`.

## 8. Визуальный стиль

База — тёмный «дизайн-инструмент» (панель-инспектор Rive/Spline):

- Канвас: `#131316`, точечная сетка `#26262b` с шагом 22 px.
- Панели/фреймы: `#1d1d21`, скругление 16–20 px; строки-контролы `#26262b`, скругление 9 px.
- Текст: основной `#f2f2f5`, вторичный `#8a8a92`, микро-заголовки капсом `#6d6d75` с трекингом.
- Акцент: зелёный `#34d36b` (активный раздел, метки фреймов, вьюпорт мини-карты, тумблеры); подложка акцента `rgba(52,211,107,.14)`, текст на подложке `#4ade80`.
- Заголовки: **Manrope** (800); основной текст — системный гротеск (Inter/Segoe UI).

Слой винтажного мультфильма (rubber-hose, 1930-е):

- Декор канваса: рисованные SVG-виньетки между фреймами — руки в перчатках, звёзды, линии движения. Монохром + редкий зелёный.
- Обводки: панели, кнопки, чипы и метки — толстый тёмный контур (2–2.5 px, `#0c0c0e`) + жёсткая смещённая «целлулоидная» тень (`5–7px 5–7px 0`) вместо мягкого блюра.
- Плёнка: зерно и 1–2 вертикальные царапины **только на фоне канваса, под панелями**, низкой прозрачности (~0.16); лёгкая виньетка по краям экрана (от 70% радиуса). На тексте, кнопках, тегах и навигации зерна нет.
- Анимации squash & stretch: кнопки сплющиваются при нажатии, фреймы «желейно» покачиваются в конце перелёта фокуса, элементы появляются с лёгким overshoot. Все анимации отключаются при `prefers-reduced-motion`.

Курсор:

- Кастомный курсор «жёлтая мультяшная рука-указатель» (изображение предоставлено) на всех интерактивных элементах: ссылки, кнопки, пункты навигации, посты, мини-карта, заголовки фреймов.
- Технически: PNG ≤ 32×32 (плюс 2x для HiDPI через `image-set`), горячая точка на кончике пальца, фолбэк `pointer`.
- Исходное изображение кладётся в `public/`, курсорные размеры генерируются на этапе подготовки ассетов.

Темы:

- По умолчанию тёмная; тумблер в навигации переключает на светлую (молочные панели `#ffffff`/`#fafafa`, те же токены в светлых значениях, тот же зелёный).
- Выбор сохраняется в `localStorage`.

## 9. Мобильная версия (< 768 px или тач-устройство без мыши)

- Канвас отключается: фреймы выстраиваются в вертикальную прокручиваемую ленту в порядке: Интро → Проекты → Блог → Резюме.
- Нижняя навигация остаётся и скроллит к разделам; мини-карта и перетаскивание отсутствуют.
- Панель поста открывается на всю ширину экрана.
- Декор (перчатки, звёзды) — точечно между секциями; зерно и виньетка сохраняются.

## 10. URL и роутинг

- Hash-роутинг (совместим с GitHub Pages): `#/about`, `#/projects`, `#/blog`, `#/resume`, `#/blog/<id>`.
- Прямой заход по якорю: вьюпорт сразу спозиционирован на нужном фрейме (без анимации перелёта), панель поста открыта при `#/blog/<id>`.
- Навигация обновляет hash; кнопки назад/вперёд браузера работают.

## 11. Технологии и деплой

- **Vite + React + TypeScript**. Стили — CSS-модули или vanilla-extract, токены в CSS-переменных.
- Markdown при сборке (unified/remark), фронтматтер — gray-matter.
- Канвас-движок пишется свой (transform-контейнер); внешние canvas-библиотеки не используются.
- Хостинг: **GitHub Pages**; деплой — GitHub Actions при пуше в main.
- Второй workflow — ежечасный cron синхронизации Telegram (см. §6).

## 12. Обработка ошибок

- Недоступен t.me-виджет (блокировка Telegram у посетителя) — в панели показываются заголовок, дата и ссылка «Открыть в Telegram».
- Ошибка парсинга в cron-синке — workflow падает с понятным логом, сайт не затрагивается.
- Некорректный `layout.json` (фрейм без координат) — фрейм размещается в резервной позиции у края раскладки, ошибка в консоль.
- Битые пути к медиа в markdown — стандартный broken-image, сборка не блокируется.

## 13. Доступность

- Вся навигация доступна с клавиатуры: Tab по интерактивным элементам, стрелки — панорамирование, `+`/`-` — зум, Esc — закрытие панели.
- Для скринридеров канвас — обычный документ: фреймы в DOM идут смысловым порядком, механика канваса — прогрессивное улучшение.
- `prefers-reduced-motion` отключает инерцию, overshoot и желейные анимации (перелёты становятся мгновенными).
- Контраст текста соответствует WCAG AA в обеих темах.

## 14. Тестирование

- Unit (Vitest): математика канваса (зум к точке, перелёт, границы), слияние blog.json при синке, парсер t.me/s (на сохранённых фикстурах HTML).
- E2E (Playwright): пан/зум/драг фрейма, перелёт по навигации, открытие поста по прямой ссылке, мобильная лента, переключение темы.
- Ручная проверка: реальный канал Telegram, тёмная/светлая темы, Firefox/Chrome/Safari.

## 15. Вне скоупа первой версии

- Комментарии, реакции, поиск, RSS.
- Сохранение пользовательской раскладки фреймов.
- Канвас на тач-устройствах.
- Админка/CMS; аналитика (можно добавить позже одной строкой).
- Мультиязычность (сайт на русском).

## 16. Нужно от владельца перед реализацией

1. Файл изображения курсора-перчатки (исходник в высоком разрешении).
2. Имя публичного Telegram-канала.
3. Тексты фреймов (можно черновые), список первых проектов, PDF резюме.
4. Домен: `<user>.github.io` или свой (для своего — добавить CNAME).
